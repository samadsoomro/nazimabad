Fix and finalize the Library Card application approval system.

- Library card applications appear in Admin Dashboard → Library Cards module

REQUIRED BEHAVIOR:
────────────────────────

1️⃣ LIBRARY CARD STATUS SYSTEM
────────────────────────

Each library card application must have a clear status:

- "pending"  → default (new applications)
- "approved" → allowed to login
- "rejected" (optional, future-safe)

Store status in database (NOT UI-only).

────────────────────────
2️⃣ ADMIN APPROVAL ACTION (FIX)
────────────────────────

When admin clicks the GREEN "Approve" button:

- Instantly update database:
  status = "approved"

- Generate / activate:
  - Library Card ID (if not already generated)
  - Login eligibility

- Disable Approve button after approval
- Change status badge from:
  PENDING → APPROVED (green)

No page reload crash
No duplicate approvals

────────────────────────
3️⃣ LOGIN RULES (CRITICAL)
────────────────────────

Library Card holders can log in ONLY if:
- status === "approved"

If status is "pending":
- Block login
- Show message:
  "Your library card application is under review."

If status is "rejected":
- Block login
- Show message:
  "Your library card application was rejected."

5️⃣ SECURITY RULES
────────────────────────
